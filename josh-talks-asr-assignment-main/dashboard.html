1f — Proposed Fixes for Top-3 Error Types

Fix 1: SpecAugment + Pitch-shift Augmentation → Phonetic Substitution

  1. Apply SpecAugment (time mask + freq mask) to all training samples
  2. Add pitch-shift augmentation ±2 semitones to create acoustic diversity
  3. Create minimal-pair training samples for confused phoneme pairs (ट/त, ड/द)
  4. Add phoneme-level CTC auxiliary loss head on the encoder output
Expected impact: 10–20% relative reduction in phonetic substitution errors.

Fix 2: Custom Vocabulary Injection → OOV / Transliteration

  1. Build bilingual loanword lexicon (top 500 English words → standard Devanagari spellings)
  2. Inject tokens via tokenizer.add_tokens() + resize model embeddings
  3. Use WFST-constrained beam search to bias towards valid Devanagari spellings
  4. Fine-tune a separate transliteration post-correction model (IndicBERT)
Expected impact: Near-elimination of OOV errors for known loanwords.

Fix 3: Token-level Loss Weighting + LM Fusion → Function Word Deletion

  1. Compute per-token frequency; assign inverse-frequency loss weights in Seq2SeqTrainer
  2. Build Hindi n-gram / IndicBERT LM; apply shallow fusion during beam search
  3. Add grammar-check post-processing step (IndicNLP) to reinsert deleted particles
Expected impact: 15–25% relative reduction in deletion errors.

Approach — 4 Combined Signals
1

Devanagari Character Validity

Reject any word containing non-Devanagari characters (Latin, stray Unicode). Lone virāma without preceding consonant → invalid. High confidence on rejection.

2

English Loanword Whitelist

Words in ENGLISH_IN_DEVANAGARI lexicon (कंप्यूटर, इंटरव्यू, …) → correct spelling, high confidence. Per transcription guideline.

3

Dictionary Lookup

Match against IndicNLP / TDIL Hindi word list. Exact match → high confidence correct.

4

Morphological Stemming

Strip known Hindi inflectional suffixes (verb forms, plural markers, adjective agreement). If stem matches → medium confidence correct.

Unreliable Categories

1. Named Entities (Proper Nouns)

People's names (Mohan, Priya), cities (Lucknow, Noida), brands — absent from dictionaries. System marks them low-confidence incorrect. Fix: integrate Hindi NER model and whitelist detected entities.

2. Long-tail Devanagari Loanwords

Rare technical terms, brand names transliterated to Devanagari (beyond our 500-word lexicon). Fix: continuously expand loanword lexicon from the dataset's own vocabulary using frequency analysis.

3. Dialectal and Archaic Words

Bhojpuri, Awadhi, Urdu borrowings absent from standard Hindi dictionaries. Fix: add dialectal word lists from CDAC / ILCI Indic corpora.
04

Lattice-Based WER Evaluation

Replace a flat reference string with a lattice of valid alternatives. Compute lattice WER to avoid penalising models for valid transcription variants.

Design Choices
WORD
Alignment unit
--
Agreement threshold
Additive
Lattice construction
Why word-level? Whisper operates at word level; character-level mixes phonetic and orthographic errors; phrase-level makes insertions/deletions hard to localise. Word-level is the standard ASR evaluation unit.

Additive lattice: We never remove the reference from a bin. We only ADD alternatives. This ensures the lattice is strictly more permissive than the reference — if the model matches the reference, its WER is unchanged.
Example: उसने चौदह किताबें खरीदीं
Reference: He bought 14 books. Five model outputs with valid alternatives.

Pseudocode — Lattice Construction + Lattice WER
ALGORITHM BUILD_LATTICE(reference R, hypotheses H, threshold K): lattice ← [LatticeBin({r_j} ∪ known_alts(r_j)) for r_j in R] for each model m in H: alignment ← word_levenshtein(R, H[m]) for (op, ref_pos, hyp_pos) in alignment: if op in {equal, substitute}: votes[ref_pos][H[m][hyp_pos]] += 1 elif op == insert: insertions[(before, after)][word] += 1 // Model agreement override for pos, votes[pos]: if count >= K: lattice[pos].add(word) // additive, never removes reference ALGORITHM LATTICE_EDIT_DISTANCE(hyp, lattice): dp[0..len(hyp)][0..len(lattice)] ← ∞ dp[0][0] ← 0; dp[i][0] ← i; dp[0][j] ← j for i, j: match_cost ← 0 if lattice[j].accepts(hyp[i]) else 1 dp[i][j] ← min(dp[i-1][j-1]+match_cost, dp[i-1][j]+1, dp[i][j-1]+1) return dp[len(hyp)][len(lattice)] LATTICE_WER(hyp, lattice, reference) = LATTICE_EDIT_DISTANCE(hyp, lattice) / len(reference)
